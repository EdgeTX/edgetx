# Getting Started with Git

[Git](https://git-scm.com/) is a widely used version control system and a backbone for EdgeTX development. GitHub is an online service hosting Git repositories. It adds to Git further collaboration features, such as issue tracking, discussions, Wikis and more.

Git supersedes earlier popular free version control systems, such as [CVS](http://cvs.nongnu.org/) and [Subversion](https://subversion.apache.org/). Git is known for it's performace, flexibility, wide acceptance and good code quality. But also for being complicated and harder to learn than many competitive products. To learn Git, you can find a brilliantly written and freely available book about Git: [Pro Git Book by Scott Chacon and Ben Straub](https://git-scm.com/book/en/v2)

If you are new to Git, then consider minimally working through the following chapters to get started working with EdgeTX Git repository:

 * [Getting a Git Repository](https://git-scm.com/book/en/v2/Git-Basics-Getting-a-Git-Repository)
 * [Branches in a Nutshell](https://git-scm.com/book/en/v2/Git-Branching-Branches-in-a-Nutshell)
 * [Rebasing](https://git-scm.com/book/en/v2/Git-Branching-Rebasing)

You can work with Git and interact with the EdgeTX repository on GitHub via command line tools, but graphical tools make the interaction especially for novices so much more easier. If you are interested on graphical user interface tools for Git, have a look for example at:

 * [TortoiseGit](https://tortoisegit.org/) (Windows only, integrates into Windows Explorer)
 * [Fork](https://git-fork.com/) (Mac & Windows)
 * [Sublime Merge](https://www.sublimemerge.com/) (Linux, Mac & Windows - no time limit evaluation)
 * [VSCode](https://code.visualstudio.com/) (Linux, Mac & Windows, code editor with basic built in Git integration - extendable with plugins like GitLens)

You can find some pointers from a Git novice perspective, getting to know Git and working with EdgeTX repository, on the following page: [Workflow for GitHub Noobs](workflow-for-github-noobs.md)
